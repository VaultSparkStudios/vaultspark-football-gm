SWOT Analysis

Strengths

  • World-state moat (owner + staff + scheme + culture)
  • PFR-calibrated realism — statistically honest simulation
  • Agent AI personalities — no competitor has this
  • Coaching tree lineage — generational scheme drift
  • Rewind ring buffer — undo safety players expect
  • TurboSim 100-season mode — dynasty play enabler
  • Premium dark aesthetic with team-color theming

Weaknesses

  • GameSession.js (5,089 ln) + app.js (6,072 ln) — monolith risk
  • Two conflicting rewind implementations (rewindManager.js vs localApiRuntime.js)
  • Test count: 3 tracked passing vs 23 test files — brittleness risk pre-beta
  • Mobile UX: single breakpoint, data density not adapted
  • No deploy live — all changes local; community can't access
  • Tutorial: 3 steps only; complexity cliff post-tutorial

Opportunities

  • Wire playerAgentAI.js to full multi-step negotiation UI
  • Surface coachingTree.js as a visual graphical UI
  • Wire beatReporter.js to a persistent news ticker
  • Rivalry DNA system — differentiated narrative memory
  • League Story Export — viral share card mechanism
  • GM Legacy Score — cross-season retention hook
  • Discord community seeding — first-mover advantage

Threats

  • Football-GM/ZenGM active dev — feature gap closes
  • localStorage quota ceiling as world-state grows
  • No beta users yet — product-market fit unvalidated
  • Growing monolith files become unmaintainable at scale
  • Beta target (2026-06-01) has 2 of 8 criteria unfinished
Areas of Improvement
  • 🔴Dual rewind implementations: rewindManager.js (async, saveStore) and localApiRuntime.js (sync, localStorage) are functionally identical but architecturally misaligned — one must be the canonical path.
  • 🔴Test coverage gap: 23 test files exist but PROJECT_STATUS reports 3 passing. Many new engine modules (coachingTree, narrativeEvents, playerAgentAI, beatReporter) have no confirmed test wiring. Pre-beta this is a reliability risk.
  • 🔴Deploy pending: All Session 3 changes are local. No live URL access for beta testers until smoke test + deploy completes.
  • 🟡Monolith growth: GameSession.js (5,089 ln) and app.js (6,072 ln) are approaching the threshold where bugs become hard to isolate and onboarding a collaborator is painful.
  • 🟡Built systems without UIs: coachingTree.js, playerAgentAI.js, beatReporter.js, dynastyTimeline.js — all complete engines with no UI surface wired yet. They exist but are invisible to players.
  • 🟡Mobile UX depth: Single 420px breakpoint handles layout collapse but doesn't adapt data density, font sizing, or interaction targets for thumb use. A GM game with 40-column stat tables needs a mobile strategy, not just a layout fix.
  • 🟡Tutorial complexity cliff: 3-step tutorial then full 53-man roster + 32-team league. No guided Year 1 challenge mode, no progressive complexity reveal.
  • 🟢Community zero: The features are ready to talk about but there's no place to do it. A Discord, a dev log page, or even a "Share Your Dynasty" button would bootstrap the community funnel.
  • 🟢No cloud save: localStorage as the only persistence layer is a retention risk — players lose everything on browser clear. IndexedDB or a lightweight server-side option would increase long-term retention.
