Top Areas of Improvement
  • Deploy NOW. Sessions 3–5 are local-only. Every feature built is invisible to real users. Deployment unlocks the Engagement and Community scores immediately — the single highest-leverage unlock remaining.
  • Run 375px smoke test. The CSS fix was shipped but never validated on a real or emulated device. 14+ new panels added in Sessions 4–5 are untested on mobile. Ship-blocking risk.
  • Refresh CURRENT_STATE.md. The context file is pre-Session 3 and describes a project with 5 fewer engine systems. AI/human onboarding from it will generate wrong assumptions.
  • Add engagement feedback loops. Persona tier unlocks fire silently. Stat leaders don't surface on Overview. Injury overlay missing. Players cannot feel progression without reactive moments.
  • Break up GameSession.js. 5,130 LOC single file is already past the "too big to reason about" threshold. DraftManager / CapManager / RosterManager split would improve testability and reduce merge complexity.
  • Add fan sentiment and world consequence. The world-state moat (owner/rival/culture) has no fan layer. Owner patience exists but fan approval is absent — a missing pillar of franchise simulation tension.
  • Close the draft system gap. No prospect comparison, no draft grade, no comp cards. The draft is the most emotionally charged moment in franchise simulation and the current surface underserves it.
  • Build a community acquisition funnel. Beta landing page + Discord invite = the first step toward community moat. Zero community spend so far despite having sharing tools, OG tags, and a newsletter ready.
